# Архитектура Chart Constructor

## Новая единая архитектура (Node.js 18 совместимая)

### Структура на сервере
```
/home/ubuntu/tchart_service/
├── node_modules/ (1.3GB - единые зависимости для всего проекта)
├── package.json (серверная конфигурация с зависимостями)
├── server.js (основной API сервер - порт 3002)
├── ecosystem.config.js (конфигурация PM2 для всех сервисов)
├── jsbot/ (только основные файлы, БЕЗ дублированных node_modules)
│   ├── tChartServerAPI.js (API рендеринга - порт 3001)
│   ├── dist/ (скомпилированные файлы)
│   └── fonts/
└── bot/ (Python боты)
    ├── tg_chart_bot.py (Telegram бот)
    ├── dc_chart_bot.py (Discord бот)
    └── dc_requirements.txt
```

### PM2 Сервисы
- **telegram-bot**: Python бот для Telegram
- **discord-bot**: Python бот для Discord  
- **tchart-server**: Node.js API сервер (порт 3002)
- **tchart-api**: Node.js API рендеринга (порт 3001)

### Ключевые изменения
1. **Убрано дублирование**: Вместо ~2.6GB зависимостей теперь 1.3GB
2. **Node.js 18**: Обновлено с 12.22.9 до 18.20.8
3. **Единые зависимости**: jsbot/ использует node_modules из родительской директории
4. **Исправлена проблема**: auto_post_channel_id больше не сбрасывается при обновлении конфигурации

### Развертывание
Используйте `deploy_new.ps1` для развертывания с поддержкой:
- Вариант 1: React приложение
- Вариант 2: Bot файлы и server.js
- Вариант 3: TChart API файлы
- Вариант 4: Установка Node.js 18 зависимостей
- Вариант 5: Полное развертывание
- Вариант 6: Только Python зависимости

### API Эндпоинты
- **Port 3001**: tChart API рендеринга (`http://server:3001/`)
- **Port 3002**: Основной API сервер (`http://server:3002/api/...`)

### Экономия ресурсов
- **Дисковое пространство**: Сэкономлено ~1.3GB
- **Память**: Уменьшение использования RAM за счет единой архитектуры
- **Управление**: Централизованная конфигурация PM2
