# Discord Chart Bot

Discord-бот для генерации красивых графиков криптовалют с автоматической публикацией.

## Возможности

- 🤖 **Slash-команды Discord** - современный интерфейс
- 📊 **Генерация графиков** - красивые графики криптовалют в реальном времени
- ⏰ **Автопостинг** - автоматическая публикация по расписанию
- 🌐 **Веб-редактор** - настройка через браузер
- 🔧 **Гибкие настройки** - полная кастомизация внешнего вида
- 👑 **Суперюзер** - расширенные возможности для администратора

## Установка

1. **Установите зависимости:**
```bash
pip install -r dc_requirements.txt
```

2. **Настройте конфигурацию:**
Отредактируйте файл `dc_config_bot.json`:
```json
{
  "bot_token": "YOUR_DISCORD_BOT_TOKEN_HERE",
  "application_id": "YOUR_APPLICATION_ID_HERE"
}
```

3. **Создайте Discord приложение:**
- Зайдите на https://discord.com/developers/applications
- Создайте новое приложение
- Получите Bot Token и Application ID
- Добавьте права бота: `Send Messages`, `Use Slash Commands`, `Attach Files`

4. **Запустите бота:**
```bash
python dc_chart_bot.py
```

## Команды

### Основные команды

- `/start` или `/help` - Показать приветствие и справку
- `/chart` - Создать график (только на серверах)
- `/settings [номер]` - Настройки бота (только для администраторов)
- `/test_api` - Тестировать подключение к API

### Команды суперюзера

- `/allmenu` - Показать все серверы (только для valikazar)

## Настройки серверов

Настройки каждого сервера хранятся в папке `dc_groups/` в формате `{guild_id}.json`.

### Основные параметры:

- **duration** - Интервал автопостинга в часах (0 = отключено)
- **network** - Блокчейн сеть (например, "polygon_pos")
- **poolAddress** - Адрес пула для мониторинга
- **interval** - Таймфрейм ("hour", "minute", "day")
- **numBars** - Количество свечей на графике

### Параметры суперюзера:

- **minimal_duration** - Минимальный интервал между постами (часы)
- **free** - Бесплатный режим (без watermark)

## Настройка через веб-интерфейс

1. Используйте команду `/settings`
2. Выберите сервер из списка
3. Нажмите "🌐 Веб-редактор"
4. Настройте внешний вид графика в браузере
5. Сохраните изменения

## Автопостинг

1. Настройте интервал через `/settings`
2. Укажите время в часах (например, 4 = каждые 4 часа)
3. Бот автоматически найдет подходящий канал для публикации
4. Для отключения установите интервал 0

## Деплой на сервер

### Автоматический деплой с помощью PowerShell скрипта

1. **Запустите скрипт деплоя:**
```powershell
.\deploy_new.ps1
```

2. **Выберите опцию деплоя:**
- `1` - React приложение
- `2` - Файлы ботов и server.js (Telegram + Discord)
- `3` - TChart API файлы
- `4` - Установка Node.js зависимостей
- `5` - Все компоненты (полный деплой)
- `6` - Только Python зависимости для ботов

3. **Что происходит при деплое:**
- Автоматическая замена токенов на деплой версии
- Создание архива с файлами ботов
- Загрузка на сервер
- Установка Python зависимостей для обоих ботов
- Создание PM2 ecosystem конфигурации
- Запуск сервисов через PM2

### PM2 сервисы на сервере

После деплоя на сервере будут запущены следующие сервисы:

- `telegram-bot` - Telegram бот
- `discord-bot` - Discord бот  
- `tchart-server` - API сервер

### Команды управления на сервере

```bash
# Просмотр статуса всех сервисов
pm2 list

# Перезапуск Discord бота
pm2 restart discord-bot

# Перезапуск Telegram бота
pm2 restart telegram-bot

# Просмотр логов Discord бота
pm2 logs discord-bot

# Остановка Discord бота
pm2 stop discord-bot

# Запуск Discord бота
pm2 start discord-bot
```

### Ручной деплой (альтернатива)

1. **Загрузите файлы на сервер:**
```bash
scp -P 2022 -r bot/ root@23.94.168.19:/home/ubuntu/tchart_service/
```

2. **Установите зависимости:**
```bash
ssh root@23.94.168.19 -p 2022
cd /home/ubuntu/tchart_service/bot
pip3 install -r dc_requirements.txt
```

3. **Настройте PM2:**
```bash
pm2 start dc_chart_bot.py --name discord-bot --interpreter python3
pm2 save
```

## Логирование

Бот записывает подробные логи всех операций:
- Генерация графиков
- Автопостинг
- Ошибки подключения
- Управление задачами

## Устранение неполадок

### Бот не отвечает на команды:
1. Проверьте, что бот онлайн
2. Убедитесь, что slash-команды синхронизированы
3. Проверьте права бота на сервере

### Не генерируются графики:
1. Проверьте подключение к tChart API (порт 3001)
2. Убедитесь, что адрес пула корректный
3. Проверьте доступность GeckoTerminal API

### Автопостинг не работает:
1. Проверьте настройку duration > 0
2. Убедитесь, что у бота есть права на отправку сообщений
3. Проверьте логи на ошибки планировщика

### Проблемы с деплоем:
1. Убедитесь, что у вас есть доступ к серверу по SSH
2. Проверьте, что PuTTY tools (pscp, plink) установлены
3. Проверьте правильность токенов в скрипте деплоя

## API зависимости

Бот использует:
- **tChart API** (localhost:3001) - для рендеринга графиков
- **tChart Web API** (localhost:3002) - для веб-токенов
- **GeckoTerminal API** - для данных о криптовалютах
